# Web UI Sitemap & Wireframes

## 목적
`web-ui` 프로젝트의 사용자 흐름을 정의하고, 화면 구성을 시각적으로 정리합니다. 이 문서는 향후 UI 개선과 구현 방향을 빠르게 공유하기 위한 초안입니다.

## Sitemap

1. **Dashboard (대시보드)**
   - 요약 카드: 총 자산 수, 위험도, 취약점 현황
   - 빠른 액션 버튼: 스캔 시작, 보고서 보기
   - 최근 스캔 상태

2. **Assets (자산 목록)**
   - 자산별 상태 요약
   - 서브도메인 / IP / 서비스 필터
   - 상태, 취약점, 노출 포트 등 주요 지표

3. **Reports (보고서)**
   - 보고서 목록 및 다운로드
   - HTML / JSON / Excel / Word 출력 옵션 안내
   - 최신 보고서 요약

4. **Scan Control (스캔 제어)**
   - 대상 도메인 입력
   - 스캔 유형 선택 (quick / standard / comprehensive)
   - 실행 상태 및 로그 요약

5. **Settings / About (설정 / 정보)**
   - Allowlist 설정 링크
   - 도구 설치 및 환경 안내
   - 프로젝트 버전 / 작성자

## Wireframes

### 1. Dashboard

- 상단 헤더: 로고, 제목, 간단 설명
- 메인 카드 영역: 4~6개 요약 카드
  - 발견 자산
  - 중대 취약점
  - 고위험 자산
  - 스캔 진행 상태
- 두 번째 영역: 최근 스캔/보고서 타임라인
- 세 번째 영역: 주요 자산 차트 또는 상태 목록

```
---------------------------------------------------------
| LOGO | ASM POC Dashboard                            |
---------------------------------------------------------
| 요약 카드 1 | 요약 카드 2 | 요약 카드 3 | 요약 카드 4 |
---------------------------------------------------------
| 최근 스캔 상태                           | 보고서 다운로드 버튼 |
---------------------------------------------------------
| 자산 상태 테이블 / 그래프 / 필터 영역               |
---------------------------------------------------------
```

### 2. Assets

- 왼쪽 필터 패널
  - 자산 유형
  - 위험도
  - 상태
- 오른쪽 목록 패널
  - 자산 이름
  - IP / 서브도메인
  - 취약점 개수
  - 웹서비스 / 포트

```
---------------------------------------
| 필터 패널 | 자산 리스트 / 상세 정보     |
---------------------------------------
|           | 자산 1                     |
|           | 자산 2                     |
|           | ...                        |
---------------------------------------
```

### 3. Reports

- 상단: 보고서 생성 버튼
- 중단: 보고서 출력 형식 선택
- 하단: 최근 생성된 보고서 리스트

```
----------------------------------------------
| 보고서 생성 버튼 | 형식 선택 (HTML, JSON, XLSX, DOCX) |
----------------------------------------------
| 최근 보고서 1                              |
| 최근 보고서 2                              |
----------------------------------------------
```

### 4. Scan Control

- 입력 필드: 타겟 도메인
- 스캔 옵션: quick / standard / comprehensive
- 실행 버튼
- 실행 로그 / 상태 배너

```
------------------------------------------
| 도메인 입력란                           |
| 스캔 유형 선택                          |
| 스캔 시작 버튼                          |
------------------------------------------
| 스캔 상태: 준비 / 진행 중 / 완료         |
| 최근 로그 출력                          |
------------------------------------------
```

## 구현 방향
- 현재 `index.html`과 `styles.css`를 기반으로 UI를 확장
- 반응형 그리드 레이아웃 사용
- 카드 스타일 + 테이블 중심 구성
- 다운로드/보고서 링크를 명확히 노출
- 모바일/태블릿에서도 읽기 쉬운 배치 유지
